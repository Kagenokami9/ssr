## 1. SCREEN 1 – START (เพิ่มเอฟเฟคข้างจอ)

เดิมหน้า START มีแค่หัวข้อ + ปุ่มตรงกลาง ด้านข้างโล่ง จึงเพิ่มเอฟเฟคประดับ (CSS ล้วน, `pointer-events: none`, อยู่ layer หลังปุ่ม):

- **ดาวเคราะห์เรืองแสงลอย** 2 ดวง — มุมล่างซ้าย (โทน cyan/nebula) และบนขวา (โทน amber/violet) ลอยขึ้นลงช้า ๆ
- **ออร์บลอยเล็ก ๆ** 4 จุด กระจายข้างจอ ลอยและจางเป็นจังหวะ
- **ขีด HUD** ข้างซ้าย-ขวา กะพริบเบา ๆ ให้ดูเป็นจอเทคโนโลยี

โครงสร้าง: `<div class="start-deco">` ครอบ `.deco-planet`, `.deco-orb`, `.deco-hud`

## 2. SCREEN 2 – WARP TRANSITION (3 วินาที + เสียง)

- ปรับให้หน้านี้ยาว **3 วินาทีพอดี** — แก้ทั้ง 2 จุดให้ตรงกัน:
  - เวลาเปลี่ยนหน้าใน `onEnter(S.WARP)`: `2700` → `3000` ms
  - ความยาว animation ใน `runWarp()`: `DUR = 2600` → `3000` ms
- **เสียง:** เล่น `เสีนงที่ 2.mp3` ตอนเข้าหน้า → หยุดก่อนเปลี่ยนไปหน้า SCAN

## 3. SCREEN 3 – SCAN READY (ปรับหน้าสแกนมือ)

ทำให้แผ่นสแกนมือ (`.hand-pad`) ดูพรีเมียมขึ้น:

- **มุมกรอบเล็ง** 4 มุมภายในแผ่น (`.pad-corner`) กะพริบเบา ๆ
- **วงเป้าเล็ง** จุดไข่ปลาหมุนช้า ๆ รอบมือ (`.hand-reticle`) พร้อมวงในสีม่วง
- เพิ่ม **แสงเรืองรอบแผ่น** เป็นค่า default (เดิมมีเฉพาะตอนแตะ)

> handler การแตะ (`id="hand-pad"`) ไม่ถูกแตะต้อง — touch/mouse ยังทำงานเหมือนเดิม

## 4. SCREEN 4 – SCANNING (7.5 วิ + ลื่นไหล + เสียง)

- **เวลา:** `CONFIG.SCAN_DURATION` `2600` → `7500` ms → progress ไล่ 0→100% ใน 7.5 วินาที
  (ค่านี้คุมทั้งเวลาหน้าจอและความยาว progress bar อยู่แล้ว)
- **ลื่นไหล ไม่สะดุด:** เขียน `startProgress()` ใหม่ จาก `setInterval` (เดินทีละ 1% แบบกระตุก)
  เป็น **`requestAnimationFrame`** ที่คำนวณ % จากเวลาจริง → เดินหน้าเนียน 60fps
- **เสียง:** เล่น `เสียงเอฟเฟคตอนสแกนมือ.mp3` ตอนเข้าหน้า → หยุดก่อนเปลี่ยนไปหน้า VIDEO (กันเสียงซ้อนกับเสียงในคลิป)

## 5. SCREEN 5 – COMPLETE (สไลด์ต้อนรับ)

ตอนจบคลิปวิดีโอ ระบบเข้าหน้านี้ (`onEnter(S.DONE)`) อยู่แล้ว จึงปรับเนื้อหาหน้าให้เป็นสไลด์ต้อนรับ:

- ข้อความ **"ยินดีต้อนรับสู่งานสัปดาห์วิทยาศาสตร์"** อยู่กลางจอ
- ใช้ฟอนต์ไทย **Kanit** (อ่านง่าย) ขนาด `clamp(28px, 5.5vw, 60px)` — พอดีกลางจอ ไม่เล็ก/ใหญ่เกินไป
- มีแสงเรือง + animation ค่อย ๆ ลอยขึ้น (`welcome-rise`)
- คง icon `id="complete-icon"` ไว้ (เปลี่ยน ✅ → 🔭) **เพราะฟังก์ชัน `refreshIcon('complete-icon')` เรียก id นี้ ถ้าลบจะ error**

## ฟอนต์ไทย (Kanit)

เดิมโหลดแค่ Orbitron + Rajdhani ซึ่ง **ไม่รองรับอักษรไทย** ข้อความไทยจึงตกไปใช้ฟอนต์ default ของ browser
จึงเพิ่ม **Kanit** เข้าไปใน Google Fonts (`&family=Kanit:wght@500;600;700`) ใช้กับข้อความต้อนรับ

## เสียงเอฟเฟค (รายละเอียดกลไก)

เสียง 2 จุดทำงานผ่านกลไกเดียวกัน:

1. `<audio id="warp-audio">` และ `<audio id="scan-audio">` (ไม่ใส่ `src` ใน HTML — กำหนดผ่าน JS จาก CONFIG แบบเดียวกับวิดีโอ)
2. คีย์ใน `CONFIG`: `WARP_AUDIO_URL`, `SCAN_AUDIO_URL`
3. ฟังก์ชันช่วย:
   - `playSound(id, url)` — รีเซ็ต `currentTime = 0` ก่อนเล่นทุกครั้ง (เล่นซ้ำได้) และห่อ url ด้วย **`encodeURI()`** เพื่อจัดการชื่อไฟล์ที่มีเว้นวรรค + อักขระไทย
   - `stopSound(id)` — หยุดและรีเซ็ตเสียง (เรียกตอนออกจากหน้า กันเสียงซ้อน)
4. เรียกใน `onEnter()` (เล่นตอนเข้าหน้า, หยุดตอนออกจากหน้า) และปุ่ม restart หยุดทั้งสองเสียงเผื่อกรณี edge

## การทดสอบ

ชื่อไฟล์เสียงมีอักขระไทย + เว้นวรรค **ต้องรันผ่าน local server** (เปิด `file://` ตรง ๆ อาจโหลดเสียงไม่ได้):

```bash
cd /home/tharinthon/Downloads/ssr
npx http-server . -p 8080
# เปิด http://localhost:8080/index.html
```

**ผลที่ตรวจแล้ว (อัตโนมัติ):**
- ✅ `index.html` + ไฟล์เสียงทั้งสอง + ฟอนต์ Kanit โหลดได้ HTTP 200
- ✅ JavaScript syntax ผ่าน (`node --check`)

**ต้องทดสอบเองใน browser (ฟังเสียง + ดูภาพจริง):**
> เครื่องนี้ไม่มี Google Chrome (มีแค่ Brave/Firefox) จึงเปิด browser อัตโนมัติผ่านเครื่องมือทดสอบไม่ได้

1. หน้า **START** → ด้านข้างไม่โล่ง มีดาวเคราะห์/ออร์บ/HUD ลอย
2. กด **Start** → หน้า WARP ~3 วินาที + ได้ยิน `เสีนงที่ 2.mp3` (หยุดเมื่อเข้าหน้า SCAN)
3. หน้า **SCAN READY** → แผ่นสแกนมือมีมุมกรอบ + วงเป้าเล็งหมุน ดูดีขึ้น
4. แตะ hand pad → หน้า **SCANNING** → progress ไล่ 0→100% เนียนใน 7.5 วิ + ได้ยิน `เสียงเอฟเฟคตอนสแกนมือ.mp3` (หยุดก่อนวิดีโอ ไม่มีเสียงซ้อน)
5. จบวิดีโอ → หน้า **COMPLETE** ขึ้น "ยินดีต้อนรับสู่งานสัปดาห์วิทยาศาสตร์" กลางจอ ฟอนต์ไทยสวยอ่านง่าย
